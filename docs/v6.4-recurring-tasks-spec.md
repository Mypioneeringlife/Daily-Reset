# The Daily Reset v6.4 Recurring Tasks Spec

## Goal
Add real recurring task support so tasks can be daily, weekly, monthly, or yearly. Tasks should only appear when they are due, and their completed/skipped/open status should reset after the correct recurrence window passes.

## Version Target
The Daily Reset v6.4 Recurring Tasks

## Required Behavior

### Daily tasks
- Appear every day.
- Reset every day according to the user's daily rollover setting.
- Existing tasks should default to daily unless otherwise changed.

### Weekly tasks
- Can be assigned to one or more weekdays.
- Only appear on the selected weekday or weekdays.
- Example: “Clean the bathroom” set to Monday appears every Monday only.
- If completed on Monday, it stays completed for that Monday.
- The next Monday, it appears open again.
- It should not clutter Tuesday through Sunday unless those days are also selected.

### Monthly tasks
- Can be assigned to a day of the month, such as the 1st, 15th, or last day.
- Only appear on their scheduled monthly date.
- Reset when the next monthly due date arrives.
- If a selected day does not exist in a month, such as the 31st in February, use the last day of that month.

### Yearly tasks
- Can be assigned to a month and day.
- Only appear on that yearly date.
- Reset when that yearly date arrives again the next year.
- For leap-day tasks, use February 28 in non-leap years unless a later option is added.

## Data Model Additions
Each task should support these fields:

```js
{
  recurrence: 'daily' | 'weekly' | 'monthly' | 'yearly',
  daysOfWeek: [1],        // weekly only; 0 Sunday through 6 Saturday
  dayOfMonth: 1,          // monthly only; 1 through 31 or 'last'
  month: 0,               // yearly only; 0 January through 11 December
  dayOfYearMonth: 1       // yearly only; day of selected month
}
```

## Completion Storage
Keep storing completion data by date in `state.days[date].items[taskId]` so the same recurring task can be completed separately for each due date.

## Filtering Logic
Add an `isTaskDueOn(task, date)` function.

Rules:
- Daily: always true.
- Weekly: true only when selected date weekday is in `task.daysOfWeek`.
- Monthly: true only when selected date matches `task.dayOfMonth`, adjusted to the month’s last day when needed.
- Yearly: true only when selected date matches the task’s month/day rule.

Then update `visibleTasks()` so it filters out tasks that are not due on the selected date.

## Task Editor UI
Add a recurrence section to the task editor:

- Repeat: Daily / Weekly / Monthly / Yearly
- Weekly: checkboxes for Sun, Mon, Tue, Wed, Thu, Fri, Sat
- Monthly: day-of-month selector, including Last day
- Yearly: month selector and day selector

## Display UI
Each task card should show the recurrence plainly:

- Daily
- Weekly on Monday
- Weekly on Monday, Thursday
- Monthly on the 1st
- Monthly on the last day
- Yearly on January 1

## Migration
During migration, every existing task should receive:

```js
recurrence: 'daily'
```

This protects all current user data and keeps v6.3 behavior unchanged until a task is edited.

## Acceptance Tests
- A weekly Monday task appears on Monday and not Tuesday.
- A weekly task completed this Monday appears open again next Monday.
- A monthly task for the 31st appears on April 30, June 30, and February 28 or 29.
- A yearly January 1 task appears only on January 1.
- Existing tasks still appear daily after migration.
- Backup and restore preserve recurrence settings.
