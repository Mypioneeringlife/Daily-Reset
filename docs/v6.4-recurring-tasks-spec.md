# The Daily Reset v6.4 Recurring Tasks Spec

## Goal
Add real recurring task support so tasks can be daily, weekly, monthly, or yearly. Tasks should only appear when they are due, and their completed/skipped/open status should reset after the correct recurrence window passes.

## Version Target
The Daily Reset v6.4 Recurring Tasks

## Required Behavior

### Daily tasks
- Appear every day.
- Reset every day according to the user's daily rollover setting.
- Existing tasks should default to daily unless otherwise changed.

### Weekly tasks
- Can be assigned to one or more weekdays.
- Only appear on the selected weekday or weekdays.
- Example: “Clean the bathroom” set to Monday appears every Monday only.
- If completed on Monday, it stays completed for that Monday.
- The next Monday, it appears open again.
- It should not clutter Tuesday through Sunday unless those days are also selected.

### Monthly tasks
- Can be assigned to a day of the month, such as the 1st, 15th, or last day.
- Only appear on their scheduled monthly date.
- Reset when the next monthly due date arrives.
- If a selected day does not exist in a month, such as the 31st in February, use the last day of that month.

### Yearly tasks and yearly reminders
- Yearly tasks should support both an event date and an optional advance reminder window.
- The event date is the real yearly date, such as a birthday, anniversary, registration renewal, holiday, annual appointment, or insurance review.
- The advance reminder controls when the task should start appearing before the event.
- Example: “Send birthday card to Mom” has an event date of Mom’s birthday and an advance reminder of 7 days. The task should begin appearing one week before her birthday so there is time to buy, write, and mail the card.
- If a yearly task has no advance reminder, it appears only on the event date.
- If a yearly task has an advance reminder, it appears from the reminder start date through the event date, unless completed earlier.
- If completed before the event date, it should remain completed for that yearly cycle and should not nag every day until the birthday arrives.
- It should reset for the next yearly cycle after the event date has passed.
- For leap-day events, use February 28 in non-leap years unless a later option is added.

## Data Model Additions
Each task should support these fields:

```js
{
  recurrence: 'daily' | 'weekly' | 'monthly' | 'yearly',
  daysOfWeek: [1],        // weekly only; 0 Sunday through 6 Saturday
  dayOfMonth: 1,          // monthly only; 1 through 31 or 'last'
  month: 0,               // yearly only; 0 January through 11 December
  dayOfYearMonth: 1,      // yearly only; day of selected month
  reminderDaysBefore: 0,  // yearly optional; 0 means event day only
  eventLabel: 'Birthday'  // optional display label such as Birthday, Anniversary, Renewal
}
```

## Completion Storage
Keep storing completion data by date in `state.days[date].items[taskId]` for normal daily, weekly, and monthly tasks.

For yearly tasks with an advance reminder window, also store a yearly cycle key so completing the task once during the reminder window counts for the whole event cycle.

Example:

```js
state.recurringCompletions = {
  'taskId:2026': {
    status: 'done',
    completedDate: '2026-04-03',
    eventDate: '2026-04-10'
  }
}
```

This prevents “Send birthday card” from reappearing every day after it was already completed for that birthday.

## Filtering Logic
Add an `isTaskDueOn(task, date)` function.

Rules:
- Daily: always true.
- Weekly: true only when selected date weekday is in `task.daysOfWeek`.
- Monthly: true only when selected date matches `task.dayOfMonth`, adjusted to the month’s last day when needed.
- Yearly without advance reminder: true only when selected date matches the task’s month/day rule.
- Yearly with advance reminder: true from `eventDate - reminderDaysBefore` through `eventDate`, unless already completed for that yearly cycle.

Then update `visibleTasks()` so it filters out tasks that are not due on the selected date.

## Task Editor UI
Add a recurrence section to the task editor:

- Repeat: Daily / Weekly / Monthly / Yearly
- Weekly: checkboxes for Sun, Mon, Tue, Wed, Thu, Fri, Sat
- Monthly: day-of-month selector, including Last day
- Yearly: event type/label, month selector, day selector, and reminder timing
- Yearly reminder timing options: Event day only, 1 day before, 3 days before, 1 week before, 2 weeks before, 1 month before, custom days before

## Display UI
Each task card should show the recurrence plainly:

- Daily
- Weekly on Monday
- Weekly on Monday, Thursday
- Monthly on the 1st
- Monthly on the last day
- Yearly on January 1
- Birthday reminder: starts 1 week before April 10
- Renewal reminder: starts 1 month before July 15

## Migration
During migration, every existing task should receive:

```js
recurrence: 'daily'
```

This protects all current user data and keeps v6.3 behavior unchanged until a task is edited.

## Acceptance Tests
- A weekly Monday task appears on Monday and not Tuesday.
- A weekly task completed this Monday appears open again next Monday.
- A monthly task for the 31st appears on April 30, June 30, and February 28 or 29.
- A yearly January 1 task appears only on January 1 when no advance reminder is set.
- A birthday-card task with a 7-day reminder appears one week before the birthday.
- If the birthday-card task is completed five days before the birthday, it stays completed and does not reappear the next day.
- The birthday-card task resets for the next yearly cycle.
- Existing tasks still appear daily after migration.
- Backup and restore preserve recurrence and yearly reminder settings.
